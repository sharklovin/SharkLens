# Security Policy

## Reporting a vulnerability

If you discover a security issue in this project, do not open a public issue with sensitive details.

Instead, share a private report with:

- a clear description of the issue
- steps to reproduce it
- potential impact
- any suggested remediation

## Safe publishing guidance

Before pushing changes to the public repository, verify that you are not exposing:

- credentials
- private datasets
- sensitive packet captures
- personal data
- internal paths or infrastructure details
